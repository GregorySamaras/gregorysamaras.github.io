Project

Azure Governance

This work focused on improving structure, visibility, and ownership across Azure environments. The aim was to move beyond ad-hoc cloud usage and make the platform easier to govern, secure, review, and operate over time.

Context

Azure environments can grow quickly, especially when different teams create resources for different projects, integrations, and workloads.

The challenge was not only technical configuration, but understanding how subscriptions, ownership, policy, cost, and security controls should fit together in a way that was maintainable.

What the work involved

  • Reviewing existing Azure subscription structure and resource organisation.
  • Looking at ownership gaps across resources, subscriptions, and workloads.
  • Considering management groups, tagging, policy, and access control as part of a more structured governance model.
  • Supporting clearer visibility into cloud resources and how they were being used.
  • Aligning governance thinking with security, compliance, operational ownership, and long-term platform maintainability.

What mattered most

The most important part was understanding that governance is not just about restricting what people can do.

Good cloud governance should make environments easier to understand, easier to secure, and easier to operate. Controls only work well when they reflect how teams actually use the platform.

This meant thinking carefully about ownership, visibility, policy impact, and where governance should guide teams rather than simply block them.

Outcome

The work helped create a clearer view of how Azure environments should be structured and governed, with more attention given to ownership, policy alignment, and operational visibility.

It also supported a more mature approach to cloud platform management, where security and governance become part of how the environment is operated rather than something added afterwards.
